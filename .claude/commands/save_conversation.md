# 保存对话总结

当用户说"结束对话"或"保存对话"时，将本次对话的关键内容保存到 `.claude/last_conversation.md`。

## 执行步骤

1. **分析对话内容**
   - 识别本次对话的主要议题
   - 提取关键发现和解决方案
   - 记录遇到的问题和解决过程
   - 总结最终结论

2. **生成结构化总结**
   包含以下部分：
   - **日期和时间**
   - **主要议题**：简短描述本次对话的核心任务
   - **关键发现**：列出重要的技术发现和洞察
   - **解决方案**：描述采用的解决方案和实现方法
   - **提交的改动**：列出所有 git commits（带 commit hash 和简短描述）
   - **当前状态**：说明已完成什么，还剩什么
   - **下一步计划**：明确列出待办事项
   - **重要文件**：列出修改或创建的关键文件
   - **注意事项**：记录需要特别注意的陷阱或限制

3. **保存到文件**
   - 文件路径：`.claude/last_conversation.md`
   - 使用清晰的 Markdown 格式
   - 包含代码片段时使用 code blocks
   - 使用表格展示对比数据

4. **压缩原则**
   - 聚焦于**可操作的信息**（不要长篇叙述）
   - 保留**关键数据**（如测试结果、页面对比统计）
   - 记录**决策理由**（为什么选择这个方案）
   - 省略**过程细节**（不记录每个尝试的中间步骤）

## 示例格式

```markdown
# 对话总结 - [日期]

## 主要议题
简短描述本次对话的核心任务（1-2 句话）

## 关键发现
- **发现 1**：描述
- **发现 2**：描述

## 解决方案
### 方案概述
简要说明采用的方案

### 实现细节
- 修改 1：位置和改动
- 修改 2：位置和改动

## 提交的改动
| Commit Hash | 描述 |
|-------------|------|
| abc1234 | 简短描述 |
| def5678 | 简短描述 |

## 当前状态
- ✅ 已完成：列表
- ⚠️ 进行中：列表
- ⏳ 待处理：列表

## 下一步计划
1. 具体任务 1
2. 具体任务 2

## 重要文件
- `path/to/file.py` - 说明
- `path/to/file.lua` - 说明

## 注意事项
- ⚠️ 注意点 1
- ⚠️ 注意点 2
```

## 注意事项

- **只保存可操作信息**：避免冗长的解释，聚焦于"做了什么"和"下一步做什么"
- **数据优先**：保留具体的测试结果、统计数据
- **决策理由**：记录"为什么"这样做（技术原因）
- **链接代码位置**：使用文件路径和行号
- **避免重复**：如果信息已在其他文档（如 MEMORY.md）中，可以引用而不重复
