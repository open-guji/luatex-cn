# 国际技术标准：CLREQ、Unicode、JLREQ、OpenType

> 本文整理与中文竖排标点相关的国际技术标准和规范，
> 为 luatex-cn 项目的排版实现提供技术参考。

## 一、W3C CLREQ（中文排版需求）

> **全称**: Requirements for Chinese Text Layout（中文排版需求）
> **地址**: https://www.w3.org/TR/clreq/ (英文) / https://w3c.github.io/clreq/zh/ (中文)
> **GitHub**: https://github.com/w3c/clreq

### 1.1 标点符号位置的地区差异

CLREQ 明确指出繁体和简体中文在标点位置上的差异：

> 「繁、簡中文的標點符號在字形上未有顯著差異，二者間的區別主要係符號的**尺寸**與**字面分布**」

| 地区 | 点号位置 | 引号层次 | 竖排引号 |
|------|----------|----------|----------|
| 港台 | **字面正中** | 先单「」后双『』 | 直角引号不变 |
| 大陆 | **字面末端/始端** | 先双""后单'' | 横排弯引号，竖排改直角引号 |

具体差异：
- **横排时**：大陆标点位于字面**左下角**；港台位于**正中**
- **竖排时**：大陆标点位于字面**右上角**；港台位于**正中**

### 1.2 标点挤压规则（Punctuation Compression）

#### 连续标点挤压

| 场景 | 挤压量 |
|------|--------|
| 结束括号 + 顿/逗/句号 | 缩减 ½ 汉字空白 |
| 顿/逗/句号 + 开始括号 | 缩减 ½ 汉字空白 |
| 连续开始括号 | 缩减 ½ 汉字空白 |
| 连续结束括号 | 缩减 ½ 汉字空白 |
| 间隔号 + 括号 | 缩减 ¼ 汉字空白 |

#### 行首行尾挤压

| 场景 | 挤压量 | 适用范围 |
|------|--------|----------|
| 行首开始括号 | 缩减始侧 ½ 汉字空白 | 通用 |
| 行尾结束括号 | 缩减末侧 ½ 汉字空白 | 通用 |
| 行尾点号 | 缩减末侧 ½ 汉字空白 | **仅简体中文** |

#### 基本压缩原则

- 两个相邻标点占 2 字宽时，压缩至最少 **1.5 字宽**
- 排版风格可进一步压缩至 **1 字宽**
- 开/关括号应与所包围的内容紧密对齐

#### 地区差异

- **大陆**: 大多数出版物应用标点挤压
- **港台**: 许多出版物使用不挤压风格

### 1.3 标点悬挂规则（Hanging Punctuation）

适用标点：**顿号、逗号、句号**

规则：将其置于前一行的行尾端、突出版心

| 限制条件 | 说明 |
|----------|------|
| 繁体中文横排 | **不做悬挂**（体例突兀） |
| 繁体中文竖排 | **可使用** |
| 连续多个标点 | **不作悬挂** |

### 1.4 行首行末禁则

**禁止出现于行首**：
- 点号（顿号、逗号、句号、冒号、分号、叹号、问号）
- 结束引号、结束括号、结束书名号
- 删节号、连接号、间隔号

**禁止出现于行尾**：
- 开始引号、开始括号、开始单双书名号

### 1.5 竖排中标点的旋转和变形

- 某些标点需**顺时针旋转90度**
- 破折号、删节号占**两个汉字空间**，不可拆至两行
- 破折号呈一直线、中间不断开
- 括注符号在直排时需依文字书写方向使用相应标注方向

### 1.6 书名号、专名号、着重号

| 标号 | 横排位置 | 竖排位置 |
|------|----------|----------|
| 书名号（甲式/波浪线） | 文字下方 | 文字**左侧** |
| 专名号（直线） | 文字下方 | 文字**左侧** |
| 着重号（圆点） | 文字下方 | 文字**右侧** |

> 相邻的书名号与专名号「须在视觉上分离予以辨别」

## 二、Unicode 竖排标准

### 2.1 UAX #50 / UTR #50：Unicode Vertical Text Layout

> **地址**: https://www.unicode.org/reports/tr50/

#### Vertical_Orientation 属性值

| 值 | 含义 | 说明 |
|----|------|------|
| **U** | Upright | 保持正立（与码表方向一致） |
| **R** | Rotated | 顺时针旋转90度 |
| **Tu** | Transformed, fallback Upright | 字体变形，无变形字形时正立 |
| **Tr** | Transformed, fallback Rotated | 字体变形，无变形字形时旋转 |

#### 关键标点的 Vertical_Orientation 值

##### CJK 标点（U+3000-U+303F）

| 码位 | 字符 | Orientation | 说明 |
|------|------|-------------|------|
| U+3000 | 　 | U | 表意空格 |
| U+3001 | 、 | **Tu** | 顿号（需要变形或正立） |
| U+3002 | 。 | **Tu** | 句号（需要变形或正立） |
| U+3008-3009 | 〈〉 | **Tr** | 单角括号 |
| U+300A-300B | 《》 | **Tr** | 双角括号 |
| U+300C-300D | 「」 | **Tr** | 单角引号 |
| U+300E-300F | 『』 | **Tr** | 双角引号 |
| U+3010-3011 | 【】 | **Tr** | 黑色透镜括号 |
| U+3014-301B | 各种括号 | **Tr** | 各种CJK括号 |
| U+301C | 〜 | **Tr** | 波浪号 |

##### 全角形式（U+FF00-FF65）

| 码位 | 字符 | Orientation | 说明 |
|------|------|-------------|------|
| U+FF01 | ！ | **Tu** | 全角叹号 |
| U+FF08-FF09 | （） | **Tr** | 全角括号 |
| U+FF0C | ， | **Tu** | 全角逗号 |
| U+FF0E | ． | **Tu** | 全角句点 |
| U+FF1A-FF1B | ：； | **Tr** | 全角冒号/分号 |
| U+FF1F | ？ | **Tu** | 全角问号 |
| U+FF3B, FF3D | ［］ | **Tr** | 全角方括号 |

##### 竖排形式（U+FE10-FE1F）

| 码位 | 字符 | Orientation | 说明 |
|------|------|-------------|------|
| U+FE10-FE19 | 竖排逗号等 | **U** | 竖排专用形式，保持正立 |

##### CJK 兼容形式（U+FE30-FE4F）

| 码位 | 字符 | Orientation | 名称 |
|------|------|-------------|------|
| U+FE30 | ︰ | U | 竖排双点前导符 |
| U+FE31 | ︱ | U | 竖排破折号 |
| U+FE32 | ︲ | U | 竖排短划线 |
| **U+FE33** | ︳ | **U** | **竖排专名号（下划线）** |
| **U+FE34** | ︴ | **U** | **竖排书名号（波浪下划线）** |
| U+FE35-FE36 | ︵︶ | U | 竖排圆括号 |
| U+FE37-FE38 | ︷︸ | U | 竖排花括号 |
| U+FE39-FE3A | ︹︺ | U | 竖排龟甲括号 |
| U+FE3B-FE3C | ︻︼ | U | 竖排黑色透镜括号 |
| U+FE3D-FE3E | ︽︾ | U | 竖排双角括号（书名号） |
| U+FE3F-FE40 | ︿﹀ | U | 竖排单角括号 |
| U+FE41-FE42 | ﹁﹂ | U | 竖排单角引号 |
| U+FE43-FE44 | ﹃﹄ | U | 竖排双角引号 |
| **U+FE45** | ﹅ | **U** | **芝麻点（着重号）** |
| U+FE46 | ﹆ | U | 白芝麻点 |
| U+FE47-FE48 | ﹇﹈ | U | 竖排方括号 |
| U+FE49-FE4F | 各种线条 | R | 上划线和下划线变体 |

> **对 luatex-cn 的意义**: U+FE33（专名号）和 U+FE34（书名号）是竖排排版中最关键的码位。

## 三、W3C JLREQ（日本排版需求）参考

> **全称**: Requirements for Japanese Text Layout
> **地址**: https://www.w3.org/TR/jlreq/ (英文+日文)
> **基础标准**: JIS X 4051（日本工业标准·日本语文书组版规则）

### 3.1 与中文竖排的比较

| 项目 | 日文（JLREQ） | 简体中文 | 繁体中文 |
|------|-------------|----------|----------|
| 点号宽度 | **半角**（加半角空白凑全角） | 全角 | 全角 |
| 点号位置 | 偏左下/偏右上 | 偏左下/偏右上 | **居正中** |
| 句号形式 | 。（空心圆，偏角落） | 。（空心圆，偏角落） | 。（空心圆，居中） |
| 引号形式 | 「」『』 | 竖排用「」『』 | 「」『』 |
| 着重号 | 圏点（字右侧） | 圆点（字右侧） | 圆点（字右侧） |

### 3.2 JLREQ 字符分类体系

JLREQ 将字符分为多个类别（character class），用于确定排版行为：

| 类别ID | 名称 | 包含字符 | 行为 |
|--------|------|----------|------|
| cl-01 | 开始括号 | （「『〈《【〔 | 半角+前空白，行末禁则 |
| cl-02 | 结束括号 | ）」』〉》】〕 | 半角+后空白，行首禁则 |
| cl-05 | 中间点 | · | 半角+前后空白 |
| cl-06 | 句号 | 。 | 半角+后空白，行首禁则 |
| cl-07 | 逗号 | 、， | 半角+后空白，行首禁则 |

### 3.3 JLREQ 悬挂标点

- 「ぶら下げ組」（hanging punctuation）技术
- 句号和逗号触发行首禁则时，可置于行尾版心外侧
- 保持网格对齐的同时避免行内调整

## 四、OpenType 竖排特性

### 4.1 相关特性标签

| 特性 | 名称 | 说明 |
|------|------|------|
| **vert** | Vertical Alternates | 提供竖排正立替代字形 |
| **vrt2** | Vertical Alternates and Rotation | vert 的超集，包含旋转字形 |
| **vrtr** | Vertical Alternates for Rotation | 提供旋转替代字形 |

### 4.2 特性对比

| 特性 | 方向 | 预旋转 | 用途 | 兼容性 |
|------|------|--------|------|--------|
| **vert** | 仅正立 | 否 | 竖排中保持正立的字形 | 与 vrtr 配合 |
| **vrtr** | 仅旋转 | 否 | 由排版引擎旋转的字形 | 与 vert 配合 |
| **vrt2** | 两者兼有 | **是** | 字体提供所有预旋转字形 | 替代 vert+vrtr |

### 4.3 设计理念

- **vert + vrtr**: 排版引擎负责图形旋转（按 UTR#50 规定）
- **vrt2**: 字体负责提供所有旋转后的字形，排版引擎不旋转

### 4.4 标点符号的 OpenType 处理

对于标点符号，字体通过 `vert` 或 `vrt2` 特性提供竖排替代字形：

```
横排字形 → vert/vrt2 → 竖排字形

, (U+FF0C) → vert → 竖排逗号（右上角/居中）
. (U+3002) → vert → 竖排句号（右上角/居中）
（ (U+FF08) → vert → ︵ (U+FE35)
《 (U+300A) → vert → ︽ (U+FE3D)
「 (U+300C) → vert → ﹁ (U+FE41)
```

### 4.5 对 LuaTeX 的影响

- LuaTeX 通过 `luaotfload` 加载 OpenType 字体
- 需要正确激活 `vert` 或 `vrt2` 特性
- 对于不提供竖排字形的标点，需要通过 Lua 代码手动调整位置

## 五、CSS Writing Modes 参考

> **地址**: https://www.w3.org/TR/css-writing-modes-4/

### 5.1 相关属性

| 属性 | 值 | 说明 |
|------|-----|------|
| `writing-mode` | `vertical-rl` | 竖排，从右到左（中文传统排版） |
| `text-orientation` | `mixed` | 混合模式：CJK正立，西文旋转 |
| `text-orientation` | `upright` | 全部正立 |
| `text-orientation` | `sideways` | 全部旋转 |

### 5.2 CSS 标点挤压

```css
/* 标点悬挂 */
hanging-punctuation: first last;

/* 标点挤压 - 提案阶段 */
text-spacing-trim: space-all;
```
