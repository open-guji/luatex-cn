# 抬头命令体系设计方案

## 一、背景知识

"抬头"是古代行文的一种规范，表示尊敬、恭敬、谦卑之意。在行文过程中，遇到需要表示尊敬的对象时，将其放到一行的顶端（竖排中即列的顶端）。

### 抬头的分类

| 类型 | 说明 | 竖排效果 |
|------|------|----------|
| **平抬** | 另起一列，顶格书写，与正常列顶端齐平 | 新列从 row=0 开始 |
| **单抬** | 另起一列，比正常列高出1格 | 新列从 row=-1 开始（伸入天头） |
| **双抬** | 另起一列，比正常列高出2格 | 新列从 row=-2 开始 |
| **三抬** | 另起一列，比正常列高出3格 | 新列从 row=-3 开始 |
| **四抬** | 另起一列，比正常列高出4格（极罕见） | 新列从 row=-4 开始 |
| **挪抬（空抬）** | 不另起列，在当前位置前空出N格 | 在当前列插入N个空格 |

### 清代科场条例规定

- **三抬**: 列祖列宗、先帝、宗庙、列圣
- **双抬**: 皇上、圣天子、圣主、御×、诏旨、孔子（先师、文宣）、国家正神（日月、五岳、关帝等）
- **单抬**: 国家、朝廷、各宫殿名称、陪祀先圣大儒、其他神灵

参考来源：[知乎 - 古文中另起一行并且高出一两个字有什么标准？](https://www.zhihu.com/question/41333967/answer/90574418)

## 二、现有实现分析

### 已有命令

| 命令 | 功能 | 实现位置 |
|------|------|----------|
| `\平抬` | 换行 + 顶格（indent=0） | `luatex-cn-core-paragraph.sty:186-202` |
| `\SetIndent{N}` | 强制设置当前行缩进为N | `luatex-cn-core-paragraph.sty:126-149` |

### 已有机制

- **三层缩进优先级**：强制缩进 > 显式缩进 > 样式栈继承
- **强制缩进编码**：`INDENT_FORCE_ZERO = -2`，`INDENT_FORCE_BASE - N = -1000 - N`
- **自动恢复**：`\SetIndent` 在 `\\` 后自动恢复到父级样式

### 不足之处

1. `\平抬` 只能顶格（indent=0），不支持单抬/双抬/三抬（负缩进，伸入天头）
2. 没有"挪抬"（不换行，空格留白）的实现
3. 没有"相对抬头"（相对当前缩进上移N格）的概念
4. 命令体系不完整，缺乏统一的设计

## 三、命令体系设计

### 3.1 核心基础命令

#### `\抬头[N]` — 基础抬头命令（绝对定位）

```
\抬头       → 等同于 \平抬，另起一列顶格书写（indent=0）
\抬头[1]    → 单抬，另起一列，比正常顶端高1格（indent=-1）
\抬头[2]    → 双抬，另起一列，比正常顶端高2格（indent=-2）
\抬头[3]    → 三抬，另起一列，比正常顶端高3格（indent=-3）
\抬头[N]    → N抬，另起一列，比正常顶端高N格（indent=-N）
```

**语义**：另起一列，文字从"天头"方向伸出N格。N=0 时等同于平抬。

**实现要点**：
- 执行 `\\` 换列
- 设置 `ATTR_INDENT` 为编码后的 `-N` 值
- 需要扩展强制缩进编码支持负值目标缩进

#### `\挪抬[N]` — 挪抬/空抬命令（不换列） 

```
\挪抬       → 在当前位置前插入1个空格
\挪抬[2]    → 在当前位置前插入2个空格
\挪抬[N]    → 在当前位置前插入N个空格
```

**语义**：不换列，在当前文字前留出N格空白，默认1格。

**实现要点**：
- 直接调用空格命令 

### 3.2 便捷快捷命令

基于 `\抬头[N]` 封装的语义化命令：

| 命令 | 等价于 | 说明 |
|------|--------|------|
| `\平抬` | `\抬头` 或 `\抬头[0]` | 另起一列，顶格书写（**已有，保持兼容**） |
| `\单抬` | `\抬头[1]` | 另起一列，高出1格 |
| `\双抬` | `\抬头[2]` | 另起一列，高出2格 |
| `\三抬` | `\抬头[3]` | 另起一列，高出3格 |
| `\空抬` | `\挪抬` 或 `\挪抬[1]` | 空1格（挪抬的别名） |

### 3.3 相对定位命令

#### `\相对抬头[N]` — 相对当前缩进上移

```
假设当前 indent=2：
\相对抬头       → 另起一列，indent=1（上移1格）
\相对抬头[2]    → 另起一列，indent=0（上移2格，等同于平抬效果）
\相对抬头[3]    → 另起一列，indent=-1（上移3格，伸入天头1格）
```

**语义**：相对于当前缩进位置上移N格。N=1 表示上移一格。

**与绝对定位的区别**：
- `\抬头[1]`（绝对）：无论当前缩进多少，始终比顶端高1格
- `\相对抬头[1]`（相对）：在当前缩进基础上上移1格

**使用场景**：当前文本已有缩进，需要在此基础上微调位置时使用。

## 四、竖排坐标系说明

```
        天头（上方留白区域）
       ┌─────────────────────┐
row=-3 │  ← 三抬文字起始位置    │
row=-2 │  ← 双抬文字起始位置    │
row=-1 │  ← 单抬文字起始位置    │
       ├─────────────────────┤ ← 正文区域上边界
row= 0 │  ← 平抬（顶格）位置    │
row= 1 │                     │
row= 2 │  ← indent=2 正文起始  │
row= 3 │                     │
  ...   │                     │
row= N │  ← 正文最后一格       │
       └─────────────────────┘
        地脚（下方留白区域）
```

- **row >= 0**：正文区域内
- **row < 0**：伸入天头区域（单抬/双抬/三抬）
- 负数 row 的文字绘制在正文区域上方的天头留白中

## 六、命令速查表（最终目标）

```latex

\抬头              % 平抬，顶格书写（indent=0）
\抬头[1]           % 单抬，高出1格（indent=-1）
\抬头[2]           % 双抬，高出2格（indent=-2）
\抬头[3]           % 三抬，高出3格（indent=-3）

% ===== 便捷命令（绝对定位） =====
\平抬              % = \抬头[0]，已有命令，保持兼容
\单抬              % = \抬头[1]
\双抬              % = \抬头[2]
\三抬              % = \抬头[3]

% ===== 相对定位抬头（另起一列） =====

% ===== 挪抬/空抬（不换列） =====
```

## 七、风险与注意事项

1. **天头空间有限**：单抬/双抬/三抬需要文字伸入天头，需确保天头留白足够，否则文字会超出页面
2. **与版心的交互**：如果版心（鱼尾、书耳等）占用天头空间，需要处理冲突
3. **向后兼容**：`\平抬` 的行为必须完全保持不变
4. **夹注中的表现**：夹注文字较小，抬头的视觉效果需要特殊处理（是否按比例缩小偏移量？）
5. **负 row 的页面边界**：需要防止文字超出物理页面边界
